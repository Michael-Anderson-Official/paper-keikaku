# 手ざわり計画表

**1週間の勉強計画を書く、紙の計画表。** 教材（科目）ごとに列を作り、日 × 教材のマスに「その日やること」を書き込む。書いた計画は同じ作者の [手ざわり手帳](https://github.com/Michael-Anderson-Official/paper-techo) の**今日のページの「勉強」欄に印刷**される。

- 公開URL: https://michael-anderson-official.github.io/paper-keikaku/
- 手ざわり手帳とセットの姉妹アプリ。手帳が「日々の顔」、計画表が「1週間の設計図」。

## コンセプト

手ざわり手帳が「予定はカレンダーに、気持ちは手帳に」なら、計画表は**「計画はここに、実行は手帳で」**。

- 計画表で1週間ぶんの勉強を俯瞰して組む（教材 × 曜日のグリッド）
- その日になると、手帳の今日のページに勉強タスクが「印刷」されて届く
- 手帳側でタスクの○（できた）をつけると、計画表のマスにも赤鉛筆の○がつく（相互に反映）

紙の質感・赤いマージン線・綴じステッチ・鉛筆の○は手ざわり手帳と共通の世界観。**UI（ヘッダー・下部ツールバー・右上☰メニュー・設定カード・フォント・縁なしモバイル表示）は手ざわり手帳とほぼ同一**に揃えてある。

## ことばづかい（おとな / こども）

手ざわり手帳と同じく、画面の文言を **「大人向け（漢字）」／「やさしい（ひらがな多め）」** で切り替えられる（設定の「ことばづかい」、キーは `pk:wording`）。**既定は大人向け**。全文言を両モードで用意している（子供向け固定にはしない）。

## 連携のしくみ（手帳と同一オリジンで localStorage 共有）

2つのアプリは同じ GitHub Pages ドメイン（`michael-anderson-official.github.io`）配下なので localStorage を共有できる。計画表は保存のたびに**連携フィード**を書き出す:

```
pk:feed:YYYY-MM-DD = {
  v: 1,
  updated: ISO日時,
  items: [ { id, subject, text, done }, ... ]
}
```

- 本文（`text` / `subject`）の**持ち主は計画表**。手帳からは書き換えない（＝手帳では計画は「印刷されたもの」で編集不可）
- ○（`done`）は手帳からも計画表からも付け外しでき、どちらが書いても相手に反映される
- 反映のトリガー: 別タブの `storage` イベント＋アプリ復帰時（`visibilitychange` / `focus` / `pageshow`）の再描画

### キー一覧

- `pk:subjects` = `[{id, name}]` … 教材（列）
- `pk:cell:YYYY-MM-DD:教材id` = 計画テキスト（計画表が持ち主）
- `pk:feed:YYYY-MM-DD` = 手帳へ渡す連携フィード（上記）
- 週の始まりは `pk:weekStart`。無ければ手帳の `pt:weekStart` に従う（設定を共有）

## 使い方

1. 右上の「＋」で教材（英語・数学・資格の教材名など）をふやす
2. 日 × 教材のマスをタップして、その日やることを書く。「できた（○をつける）」も切り替えられる
3. 教材の見出しをタップで改名・列ごと削除
4. 下部ツールバーの ◀前へ／次へ▶ で週送り、「今週」で今週へ戻る
5. 右上「☰」→「設定」で ことばづかい・かみ（テーマ）・週の始まり・データのひかえ（バックアップ/復元）。「手ざわり手帳をひらく」で姉妹アプリへ

## テーマ（かみ）— 手帳と共有

アイボリー／白／黒／黒板／電光掲示板の5種類。**手ざわり手帳と共通の設定**で、選ぶと紙の色・罫線・文字色が連動する。黒板はチョーク字（Yomogi＋かすれマスク）、電光掲示板はドット字（DotGothic16＋発光）と、手帳と同じ質感まで再現。

- 計画表のテーマは `pk:theme`。未設定なら手帳の `pt:paper` に従う（週の始まりと同じ共有パターン）
- どちらのアプリで変えても、`storage` イベント／アプリ復帰時に相手側へ反映

## 技術

- 単一 `index.html`・vanilla JS・依存なし
- フォント（Klee One / Shippori Mincho）は手ざわり手帳と同じく `fonts/` に自前ホスト（Google Fontsの実配信ファイルをそのまま取得・オフライン動作。ライセンスは `fonts/OFL*.txt`）
- アイコン（`apple-touch-icon.png` / `icon-192.png` / `icon-512.png`）は紙・赤マージン線・計画グリッド・赤鉛筆の○（できたマーク）をモチーフにしたオリジナルデザイン。手帳の実写風アイコンとは絵柄が異なる（手描き風ではなく線画）
- `manifest.webmanifest` ＋ `sw.js` でオフライン対応（PWA）
- iPhone Safari 想定。スマホでは縁なし全画面

## ネイティブ化（Capacitor・手帳と同じ流儀）

App Store 配布用に Capacitor で iOS アプリとして包める（`appId: jp.tezawari.keikaku`）。中身の web 資産はそのまま `www/` にコピーするだけで、作り直しは不要。

```
npm install               # Capacitor CLI / core / ios を入れる（初回のみ）
npm run build-www         # index.html などを www/ にコピー
npx cap sync ios          # www/ を iOS プロジェクトへ反映（= npm run sync でまとめて実行）
# 以降は Mac + Xcode で ios/App を開いてビルド・提出
```

- iOS プロジェクトは `ios/`（Capacitor 8 = Swift Package Manager 構成。CocoaPods 不要）。アプリアイコンは `ios/App/App/Assets.xcassets/AppIcon.appiconset/` に 1024×1024 を設置済み
- `node_modules/`・`www/`・`ios/App/App/public/` は生成物のため git 管理外（`.gitignore`）
- **ビルド・提出は Mac（Xcode）と Apple Developer Program（年 $99）が必要**。この作業は開発者の Mac 側で行う
- **注意（連携の制約）**: 手帳との localStorage 共有は「同一オリジンのブラウザ」でのみ成立する。ネイティブアプリ同士は別サンドボックスになるため、Web 版どうしの連携はネイティブでは効かない。ネイティブで連携させるには App Group や共有ファイル等の別実装が必要（今後の課題）
