# App Privacy（プライバシー"配慮"ラベル）申告メモ — 手ざわり計画表

App Store Connect の「App Privacy」質問票で選ぶ内容の下書き。**最終判断・提出はユーザー自身が行うこと**（開発者の自己申告であり、Appleの審査でも実際の挙動と食い違うと差し戻される）。

## データ収集の有無

このアプリはデータを収集します（以下2種類）。利用統計（アナリティクス）は現状実装していないため申告不要。

### 1. ISBN（書名・表紙の自動取得を使った場合のみ）

- **データの種類**: 「その他のデータ」→「その他のデータタイプ」（Other Data Types）
- **内容**: バーコードで読み取った本のISBN番号のみ。openBD／Google Booksへ書名・表紙を問い合わせるために送信
- **ユーザーに関連付けられるか**: **いいえ** — ISBNは本の識別子であり利用者個人とは無関係。氏名・アカウント情報は送らない
- **トラッキングに使用**: **いいえ**
- **用途**: 「App の機能（App Functionality）」
- 補足: 使わなければ（手入力・写真から表紙設定）この通信自体発生しない

### 2. 勉強計画の連携データ（連携コード使用時のみ）

- **データの種類**: 「ユーザーコンテンツ」→「その他のユーザーコンテンツ」（Other User Content）
- **内容**: 日付・教材名・やることの文面・できた○（姉妹アプリ「手ざわり手帳」との同期用）
- **ユーザーに関連付けられるか**: **いいえ** — 連携コード（ランダムな文字列）でしか紐付かず、氏名・メール等とは結び付けない
- **トラッキングに使用**: **いいえ**
- **用途**: 「App の機能（App Functionality）」

## 収集しないデータ（申告不要）

- 計画表に書いた教材名・やること・できた○そのもの（連携フィードに載る一部を除き端末内のみ）
- 目次OCRで撮影した画像・読み取ったテキスト（Tesseract.jsで端末内完結、外部送信なし）
- 表紙写真（写真から設定した場合は端末内保存のみ。openBD経由の表紙も端末内、Google Books経由のみ表示のたび外部読み込みだが個人データではない）
- 位置情報・連絡先・健康情報・財務情報 等

## 備考

- 広告・IAPは初回リリースに含めないため、広告SDK関連の申告（IDFA/ATT等）は今回は不要。実装時に本ファイルを更新すること。
- Cloudflare Workers（keikakuchou-notify）が中継サーバーだが、開発者自身が管理するインフラであり第三者への提供・販売はしていない旨、必要なら「第三者と共有しない」を選択。
- Google Books側への表紙画像読み込みは「リンク先の第三者サービス」の扱いになる可能性がある点、審査で指摘されたら「第三者パートナーのSDK/サービス」欄への追記を検討。
